# Техническое задание

## Наименование системы
Разработка информационной системы автоматизации подбора материалов для ремонта помещений

---

## Обоснование необходимости разработки

Обоснование для такой системы строится на том, что традиционный подбор материалов — это низкая эффективность из ручных расчетов, избыточных закупок и бесконечных поездок по магазинам.

Внедрение системы позволит достичь следующих эффектов:

1. Экономическое обоснование (Бизнес-эффект) Рост среднего чека (Upsell/Cross-sell): Система автоматически предлагает сопутствующие товары (грунтовку к краске, крестики к плитке, клей к обоям). Покупатель получает полную комплектацию заказа в одном месте, что исключает необходимость докупать расходные материалы у других поставщиков.

2. Снижение процента возвратов: Точный расчет исключает ситуацию приобретения избыточного количества материалов («остатков»). Это избавляет пользователя от необходимости заниматься возвратом товара, а магазин — от лишних логистических операций.

3. Оптимизация временных затрат: Автоматизация процесса подбора позволяет сократить время на составление полной спецификации материалов с нескольких часов до нескольких минут. Пользователю не требуется самостоятельно изучать инструкции по расходу и совместимости строительных смесей.

4. Минимизация технологических ошибок: Использование программных алгоритмов исключает «человеческий фактор» при расчете сложных поверхностей и подборе сопутствующих товаров. Система гарантирует, что выбранный клей, грунтовка или фурнитура технически подходят к основному материалу отделки.

5. Прозрачность бюджета ремонта: Формирование единого списка материалов позволяет заранее определить стоимость ремонта.

Внедрение данной системы позволит перевести процесс подготовки к ремонту из области примерных оценок в плоскость точных расчетов. Полученная автоматизированная спецификация становится для клиента универсальным инструментом для закупки материалов и контроля качества сметных работ.

---

## Назначение и цели разработки

### Назначение
Информационная система предназначена для автоматизации процесса расчета необходимых строительных и отделочных материалов на основе геометрических параметров помещений и выбранных типов отделки. Система может использоваться как частными лицами для самостоятельного планирования ремонта, так и специалистами (прорабами, менеджерами магазинов) для формирования профессиональных спецификаций.

### Цели
1. Повышение точности расчетов: Обеспечение математически верного расчета объемов материалов с учетом коэффициентов на обрезку, нахлест и запас (стандартно 5–10%).
2. Исключение технологической несовместимости: Автоматическая проверка возможности совместного применения различных материалов (например, соответствие типа клея и веса плитки).
3. Создание единого стандарта спецификации: Формирование структурированного списка товаров, включающего артикулы, количество, вес и необходимые сопутствующие инструменты.
4. Информационная поддержка пользователя: Предоставление рекомендаций по выбору расходных материалов (грунтовки, крестики, гидроизоляция), которые необходимы для соблюдения технологии работ.

## Функциональные требования системы (по ролям)

### 1. Функции пользователя (клиента)

1. Пользователь должен иметь возможность регистрации в системе.
2. Пользователь должен иметь возможность входа в систему (аутентификация).
3. Пользователь должен иметь возможность выбора типа помещения для подбора материалов.
4. Пользователь должен иметь возможность ввода параметров помещения (длина, ширина, площадь, периметр) с проверкой корректности данных.
5. Пользователь должен иметь возможность выбора уровня ремонта (эконом, стандарт, премиум).
6. Пользователь должен иметь возможность указания бюджета для подбора материалов.
7. Пользователь должен иметь возможность просмотра каталога строительных и отделочных материалов.
8. Пользователь должен иметь возможность поиска материалов по названию.
9. Пользователь должен иметь возможность фильтрации материалов по характеристикам (тип, цена, назначение, производитель).
10. Пользователь должен иметь возможность просмотра детальной информации о материалах (описание, характеристики, изображения, цена).
11. Пользователь должен иметь возможность сравнения нескольких материалов.
12. Пользователь должен иметь возможность выбора и формирования списка материалов.
13. Пользователь должен иметь возможность получения нескольких вариантов подбора материалов.
14. Пользователь должен иметь возможность получения рекомендаций по альтернативным материалам.
15. Пользователь должен получать уведомление при превышении бюджета.
16. Пользователь должен получать предупреждения о несовместимости материалов.
17. Пользователь должен иметь возможность сохранения результатов подбора.
18. Пользователь должен иметь возможность просмотра истории своих расчетов.
19. Пользователь должен иметь возможность редактирования ранее сохраненных подборов.
20. Пользователь должен иметь возможность экспорта списка материалов (сметы).

---

### 2. Функции администратора

1. Администратор должен иметь возможность входа в систему.
2. Администратор должен иметь возможность добавления материалов в каталог.
3. Администратор должен иметь возможность редактирования материалов.
4. Администратор должен иметь возможность удаления материалов.
5. Администратор должен иметь возможность управления категориями материалов.
6. Администратор должен иметь возможность управления производителями и характеристиками материалов.
7. Администратор должен иметь возможность загрузки изображений материалов.
8. Администратор должен иметь возможность просмотра статистики использования системы.
9. Администратор должен иметь возможность просмотра популярности материалов.
10. Администратор должен иметь возможность управления пользователями (при необходимости).

---

### 3. Функции системы (автоматические процессы)

1. Система должна обеспечивать аутентификацию и авторизацию пользователей.
2. Система должна обеспечивать разграничение прав доступа между пользователем и администратором.
3. Система должна автоматически рассчитывать площадь помещения на основе введённых данных.
4. Система должна автоматически рассчитывать необходимое количество материалов с учетом характеристик и коэффициента запаса.
5. Система должна обеспечивать подбор оптимального количества материалов с минимальными остатками.
6. Система должна учитывать тип помещения при подборе материалов.
7. Система должна учитывать тип поверхности (стены, пол, потолок).
8. Система должна рассчитывать итоговую стоимость материалов.
9. Система должна учитывать бюджет пользователя при подборе.
10. Система должна уведомлять о превышении бюджета.
11. Система должна проверять совместимость материалов.
12. Система должна предупреждать о несовместимости материалов.
13. Система должна формировать рекомендации по альтернативным материалам.
14. Система должна формировать несколько вариантов подбора материалов.
15. Система должна формировать рекомендации по последовательности применения материалов.
16. Система должна обеспечивать хранение данных о пользователях, материалах и расчетах в базе данных.
17. Система должна обеспечивать сохранение и восстановление результатов подбора.
18. Система должна формировать отчеты по активности пользователей и популярности материалов.

## Нефункциональные требования

1. Производительность:  
• Время отклика системы при загрузке страниц и выполнении расчетов не должно превышать 2–3 секунд при одновременной работе до 100 пользователей.

2. Доступность:  
• Система должна быть доступна пользователям не менее чем 99% времени в месяц.

3. Масштабируемость:  
• Архитектура системы должна позволять добавление новых типов материалов, категорий и функций без изменения основной логики приложения.

4. Безопасность:  
• Доступ к административной панели должен осуществляться только после прохождения процедуры аутентификации и авторизации.

5. Конфиденциальность:  
• Персональные данные пользователей должны храниться в защищенном виде и не передаваться третьим лицам.

6. Надёжность:  
• При сбоях в работе системы данные пользователей и результаты расчетов должны сохраняться и восстанавливаться из резервных копий.

7. Удобство интерфейса:  
• Интерфейс системы должен быть интуитивно понятным, простым в использовании и адаптированным под различные устройства.

8. Совместимость:  
• Система должна корректно работать в современных браузерах: Google Chrome, Mozilla Firefox, Microsoft Edge и Safari.

9. Время обновления данных:  
• Обновленная информация о материалах (цены, характеристики) должна отображаться пользователю не позднее чем через 60 секунд после изменения.

10. Логирование:  
• Система должна вести журнал действий пользователей, включая дату, время, тип действия и измененные данные.
